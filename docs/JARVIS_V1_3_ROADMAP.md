# Jarvis v1.3 Development Roadmap

**Start Date**: 2026-06-30  
**Baseline Version**: `1.2.0`  
**Target Branch**: `v1.3-development`  

---

## 🎯 v1.3 Release Target Goals

### 1. 🖥️ Advanced HUD
- Move from compact/expanded states to full voice waveform controls overlays.
- Dynamic drag-and-drop boundary locks.
- Real-time approvals prompts directly inside floating HUD layout.

### 2. 🕒 Scheduled Monitoring
- Transition watchlist monitoring to cron-like task intervals.
- Low-power background check daemons on active watchlist.
- Smart notification card alerts.

### 3. 🌐 WordPress & SEO Automation
- Sandbox execution modules for scanning site SEO health.
- WordPress posts content draft reviews and listings verification.
- Enforce strict sandboxed command guards on remote configurations.

### 4. 🚀 App Release Workflow Upgrades
- Step-by-step assistant guiding manual store publishing.
- Fastlane configuration checks and automatic certificate audits.
- Privacy policy manifest logs compiler.

### 5. 📴 Local Lightweight Fallback
- Dynamic detection of internet network link outages.
- Fallback to fully local lightweight command parser model for core features.

### 6. 🔗 Multi-step Task Chains
- Orchestrate sequential actions (e.g. read email -> draft reply -> create issue -> commit code) in single voice requests.

### 7. 🔌 Plugin System Upgrades
- Sandbox rules extensions.
- Dynamic plugins loading from local folders.
- Health reports and activity logging dashboards.

### 8. ⚡ Performance Optimization
- Reduce frontend React view bundle sizes.
- Optimize telemetry SQLite SQLite command indices.
- Cap RAM overhead under 250 MB on 8GB Mac device.

### 9. 🔒 Security Hardening
- Keychain authentication gates.
- Stricter inputs scans to redact API credentials.
- Block remote shell executions.
